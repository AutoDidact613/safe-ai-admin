1. Branch (ענף)
הסבר: ענף הוא עותק פרטי של הפרויקט שבו אנחנו עובדים בנפרד, כדי לא להרוס את הקוד הראשי.
מתי משתמשים: כשרוצים להוסיף פיצ'ר חדש.

2. Pull Request (PR - בקשת מיזוג)
הסבר: בקשה רשמית לחבר את השינויים שעשינו בענף שלנו אל תוך הענף הראשי.
מתי משתמשים: כסיימנו לעבוד ואנחנו רוצים שהקוד ייכנס לפרויקט המשותף.

3. Conflicts (קונפליקטים)
הסבר: מצב שבו המחשב לא יודע איזה קוד לשמור, כי שני אנשים שינו את אותן שורות בדיוק.
מתי משתמשים: צריך לפתור את זה ידנית ולבחור את הגרסה הנכונה.
